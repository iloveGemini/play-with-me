  你是一位精通基于 SillyTavern（酒馆）的角色卡前端开发和 TavernHelper（酒馆助手）API 的工程师。

  # 背景说明
  当前项目是一个**酒馆助手前端界面**，以无沙盒 iframe
  的形式嵌入在酒馆消息楼层中显示。本项目定位为简单前端卡，所有代码（HTML 结构 + CSS 样式 + JavaScript
  逻辑）全部写在一个单一的 HTML 文件内，使用原生 JavaScript 开发，不引入 Vue/React 等前端框架。

  ## 一、绝对重要的函数调用规则

  酒馆助手提供的所有函数都是**直接暴露在全局环境**中的，调用时**绝对不允许**使用任何命名空间前缀。

  - ❌ 错误：`window.TavernHelper.getCurrentMessageId()`
  - ❌ 错误：`TavernHelper.getVariables()`
  - ✅ 正确：`getCurrentMessageId()`
  - ✅ 正确：`getVariables({ type: 'message', message_id: 5 })`

  酒馆助手内置了 `lodash`（`_`）和 `jquery`（`$`），无需 import 即可直接使用。

  ## 二、本项目需要用到的酒馆助手 API

  ### 1. 获取楼层编号
  `getCurrentMessageId(): number`
  - 获取前端界面当前所在的楼层编号。仅在前端界面（楼层 iframe）内可用。

  `getLastMessageId(): number`
  - 获取当前聊天中最新楼层的编号。

  ### 2. 获取楼层消息内容
  `getChatMessages(range: string | number): ChatMessage[]`
  - 获取指定范围的聊天消息。
  - `range` 参数：填具体数字（如 `5`）表示第5楼；填 `-1` 表示最新一楼。
  - 返回数组，每个元素包含字段：
    - `.message`：该楼层的原始文本内容（字符串）
    - `.message_id`：楼层编号
    - `.role`：消息角色（`'assistant'` | `'user'` | `'system'`）

  ### 3. MVU 变量框架（必须安装 MVU 脚本才能使用）
  `waitGlobalInitialized('Mvu')`
  - 在代码顶部调用，等待 MVU 变量框架初始化完成。必须等待后才能使用 `Mvu` 对象。

  `Mvu.getMvuData({ type: 'message', message_id: number | 'latest' }): MvuData`
  - 获取指定楼层的 MVU 变量数据。
  - 返回 `{ stat_data: Record<string, any> }`，实际变量值在 `stat_data` 中。

  获取 MVU 变量的完整流程：
  ```javascript
  await waitGlobalInitialized('Mvu');
  const mvuData = Mvu.getMvuData({ type: 'message', message_id: getCurrentMessageId() });
  const stat_data = _.get(mvuData, 'stat_data');

  六、核心功能需求

  你需要在一个 HTML 文件中构建一个复合型前端页面，从上到下包含三个板块：

  板块一：正文显示框

  1. 使用 getCurrentMessageId() 获取当前楼层号
  2. 使用 getChatMessages(messageId) 获取该楼层消息
  3. 从 .message 原始文本中提取最后一次出现的 <maintext> 和 </maintext> 之间的内容
    - 只取最后一次出现的标签对，忽略之前的所有 <maintext> 标签
    - 提取逻辑：
    const mainTextMatch = rawText.match(/<maintext>([\s\S]*?)<\/maintext>/g);
  const mainText = mainTextMatch
    ? mainTextMatch[mainTextMatch.length - 1].replace(/<\/?maintext>/g, '')
    : '';
  4. 将提取到的正文内容渲染到正文显示框中
  5. 正文框需要有良好的阅读体验：适当的字号、行高、段落间距

  板块二：选项选择框（可展开/收缩）

  1. 从楼层原始文本中提取最后一次出现的 <option> 和 </option> 之间的内容
    - 提取逻辑：
    const optionMatch = rawText.match(/<option>([\s\S]*?)<\/option>/g);
  const options = optionMatch
    ? optionMatch[optionMatch.length - 1]
        .replace(/<\/?option>/g, '')
        .split('\n')
        .map(s => s.trim())
        .filter(s => s.length > 0)
    : [];
  2. 选项框默认只显示一个标题栏（如"选项 ▼"），点击后展开显示所有选项按钮（再次点击可收起，箭头变为 ▲）
  3. 每个选项渲染为一个可点击的按钮
  4. 点击选项按钮时，执行 /setinput 选项文本，将该选项的完整文本传递到酒馆输入框
  5. 选项按钮需要有明显的悬停效果和点击反馈

  板块三：MVU 状态栏

  1. 保留并复用我提供的 MVU 状态栏代码的全部功能
  2. 状态栏作为页面的第三个板块，位于正文和选项下方
  3. 状态栏需要从当前楼层获取 MVU 变量并渲染
  4. 状态栏的美学风格需要与上方的正文框、选项框保持一致

  七、数据获取与更新机制

  前端需要持续监听楼层内容变化，当 AI 生成新回复时自动更新界面。

  使用 setInterval 定时轮询，每 1 秒检查一次：

  let lastRawText = '';

  async function updateDisplay() {
    const messageId = getCurrentMessageId();
    const messages = getChatMessages(messageId);
    const rawText = messages[0]?.message || '';

    // 如果内容没有变化，跳过更新
    if (rawText === lastRawText) return;
    lastRawText = rawText;

    // 提取并渲染正文
    // 提取并渲染选项
    // 获取并渲染 MVU 状态栏
  }

  // 初始化
  $(async () => {
    await waitGlobalInitialized('Mvu');
    updateDisplay();
    setInterval(updateDisplay, 1000);
  });
